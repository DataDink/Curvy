Application

If you're curious about Curvy and just getting started you will probably want to know a little bit about how it is structured.

Curvy, by itself, is a host for modules and services that come together to form an application. Calling the Curvy.create() method will create a new instance of Curvy and attach itself to your page.

Like Angular you can pre-load any services, bindings, and viewmodels required to run your application. With Curvy, however, you can also load up additional functionality at any time.

Pre-Configuration

Your application will likely be comprised of a number of services, custom bindings, and view-models. These can all be configured directly from the register function on the "Curvy" constructor.

Services Curvy.register.service(name, [dependencies], constructor)
Allows you to register a new service for your application.
A service is a constructor that will be created only once per application and be made available to any other part of your application by name.

Injection Constructors
An injection constructor (as in Angular) is a regular javascript constructor at the end of an array of strings that define the required dependency values for a new object.
Each string preceeding the constructor in the array defines a dependency by name that will be passed to the constructor in the order specified.
Unlike Angular, Curvy requires that dependencies be defined with strings instead of looking at the parameter names in the constructor. This decision was made to avoid issues commonly caused through processes like minification that will alter the names of parameters in functions.

Curvy.register.instance(name, object)
Registers a pre-constructed object as a named dependency.

Curvy.register.singleton(name, constructor)
Registers an injection constructor as a named dependency. This will not be constructed until it is needed for the first time as a dependency in the application. Singleton dependencies are constructed once per application and re-used for all other dependency requirements.

Curvy.register.contextual(name, constructor)
Registers an injection constructor as a named dependency. This will not be constructed until it is needed for the first time as a dependency in the application. Contextual dependencies are constructed once per resolution scope and re-used for all descendant dependency requirements.

Curvy.register.transient(name, constructor)
Registers an injection constructor as a named dependency. This will not be constructed until it is needed for the first time as a dependency in the application. Transient dependencies are always re-constructed every time they are required as a dependency.

Services Curvy.register.service(name, constructor)
Provides a method to add custom service to your application. Registration of services are similar to singleton services in that they are created only once and recycled each time they are needed.

Bindings Curvy.register.binding(attributeName, constructor)
Provides a method to add custom bindings to your application. Registration of bindings is similar to transient services in that they are newly created each time they are needed, but bindings can't be asked for as dependencies. Bindings are registered by their attribute name and will only be triggered with a matching attribute in the DOM.

View Models Curvy.register.viewmodel(name, constructor)
Provides a method to register view models for your application. Registration of view models is similar to transient services, but like bindings can't be asked for as dependencies. View models can only be triggered when a "view-model" attribute is added to the DOM with a value matching the registered name.

Post-Configuration

After the application has been instantiated it is still possible for additional services and view models to be added to your application "on the fly".

Services The constructed application (Curvy.create()) provides instance, singleton, contextual, and transient functions for late registration of services.

(Curvy.create()).dependencies.register.instance(name, object)
registers a pre-constructed object as a named dependency.

(Curvy.create()).register.singleton(name, constructor)
Registers an injection constructor as a named dependency. This will not be constructed until it is needed for the first time as a dependency in the application. Singleton dependencies are constructed once per application and re-used for all other dependency requirements.

(Curvy.create()).register.contextual(name, constructor)
Registers an injection constructor as a named dependency. This will not be constructed until it is needed for the first time as a dependency in the application. Contextual dependencies are constructed once per resolution scope and re-used for all descendant dependency requirements.

(Curvy.create()).register.transient(name, constructor)
Registers an injection constructor as a named dependency. This will not be constructed until it is needed for the first time as a dependency in the application. Transient dependencies are always re-constructed every time they are required as a dependency.

View Models (Curvy.create()).viewmodel(name, constructor)
Provides a method to register view models for your application. Registration of view models is similar to transient services, but like bindings can't be asked for as dependencies. View models can only be triggered when a "view-model" attribute is added to the DOM with a value matching the registered name.
