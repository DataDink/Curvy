Application

If you're curious about Curvy and just getting started you will probably want to know a little bit about how it is structured.

When adding Curvy to your page you are given a single global constructor: "Application". The Application constructor allows you to pre-configure your application if you would like or you can wait until after construction to load in your client code. The only thing you are required to preload are your custom bindings. see Bindings

Pre-Configuration

Your application will likely be comprised of a number of services, view-models, custom bindings, and configurations. These can all be configured directly from the extend function on the "Application" constructor.

Configurations The "Application.extend(constructor)" method accepts an injection constructor that will be run when the "Application" is instantiated. You are provided normal injection service for this constructor but constructed configuration object will be inaccessible to the rest of the application.

Services Services can be added as dependencies for the rest of your applications by using the registration functions located on the "Application.extend.register" object. All services added here are added by name as injection contructors except when added as an instance.

Injection Constructor
An injection constructor is a function which asks for dependencies by name. If a service exists that matches a parameter name in the function's declaration it will be passed when the injection constructor is being instantiated. Injection constructors are treated as javascript class definitions and are constructed using the "new" keyword.
An injection constructor can also be wrapped in an array which precedes the constructor function with dependency names as strings to avoid minification issues.

Application.extend.register.instance(name, object)
registers a pre-constructed object as a named dependency.

Application.extend.register.perApp(name, constructor)
registers an injection constructor as a named dependency. This will not be constructed until it is needed for the first time as a dependency in the application. PerApp dependencies are constructed once per application and re-used for all other dependency requirements.

Application.extend.register.perScope(name, constructor)
registers an injection constructor as a named dependency. This will not be constructed until it is needed for the first time as a dependency in the application. PerScope dependencies are constructed once per resolution scope and re-used for all descendant dependency requirements.

Application.extend.register.perResolve(name, constructor)
registers an injection constructor as a named dependency. This will not be constructed until it is needed for the first time as a dependency in the application. PerResolve dependencies are always re-constructed every time they are required as a dependency.

Bindings Custom bindings can be added to the application using the "Application.extend.binding(attribute, constructor)" registration function. Bindings are similar to perResolve services in that they are newly constructed each time they are needed, but bindings can't be asked for as dependencies and are only triggered when the specified attribute name is found in the DOM.
